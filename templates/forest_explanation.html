Random Forest Explanation

Machine Learning Model for Public Lighting Failure Risk Prediction

1. What is Random Forest?

Random Forest is a supervised machine learning algorithm based on the concept of ensemble learning. Instead of relying on a single decision tree, it builds a large collection of trees and combines their predictions to produce a more accurate and stable result.

In this project, Random Forest classifies Bogotá's public lighting infrastructure into:

  • Low Failure Risk
  • Medium Failure Risk
  • High Failure Risk

2. How Does Random Forest Work?

The algorithm creates multiple decision trees during training, each built on a random subset of the data and features. The final prediction is made by aggregating the votes of all trees.

Final Prediction = Majority Vote (Tree₁, Tree₂, ..., Treeₙ)

This process involves two key concepts:

  • Bagging: Each tree is trained on a random sample of the data with replacement.
  • Feature randomness: Each split considers only a random subset of features, reducing correlation between trees.

3. Variables Used in the Model

The model uses infrastructure variables extracted from Bogotá's monthly public lighting records.

Variable Description
% LED Percentage of LED infrastructure per locality
TOTAL Total public lighting points per locality

4. OOB Score — A Unique Feature of Random Forest

Random Forest includes a built-in validation mechanism called Out-Of-Bag (OOB) Score. Since each tree is trained on a random subset, the remaining samples (not used in training) can be used to estimate model accuracy without needing a separate validation set.

  • OOB Score close to 1.0 indicates strong generalization.
  • It serves as an unbiased estimate of model performance.
  • Useful when the dataset is small, as no data is wasted on validation.

5. Application in This Project

Random Forest is applied to estimate failure risk in Bogotá's public lighting infrastructure using LED adoption rates and total luminaire counts.

The model helps identify:

  • Critical lighting zones with high failure probability
  • Areas with outdated or mixed technology infrastructure
  • Potential preventive maintenance priorities
  • Zones with strong LED modernization progress

6. Advantages of Random Forest

  • Robust against overfitting due to tree averaging
  • Handles non-linear relationships naturally
  • Provides feature importance scores
  • Built-in OOB validation without extra data splitting
  • Works well with small and medium-sized datasets

7. Model Limitations

  • Slower prediction compared to simpler models
  • Less interpretable than a single decision tree
  • Memory-intensive when using many trees
  • Risk labels are estimated rather than obtained from real failures

8. Conclusion

Random Forest provides a robust and reliable approach for estimating public lighting failure risk. By combining hundreds of decision trees, the model reduces variance and produces stable predictions even with limited data.

Its OOB score and feature importance metrics make it especially valuable for infrastructure analysis, supporting data-driven maintenance decisions for Bogotá's public lighting system.
