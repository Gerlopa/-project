LightGBM Explanation

Machine Learning Model for Public Lighting Failure Risk Prediction

1. What is LightGBM?

LightGBM (Light Gradient Boosting Machine) is a high-performance gradient boosting framework developed by Microsoft. It is designed to be faster and more memory-efficient than traditional boosting algorithms while maintaining high accuracy.

In this project, LightGBM classifies Bogotá's public lighting infrastructure into:

  • Low Failure Risk
  • Medium Failure Risk
  • High Failure Risk

2. How Does LightGBM Work?

LightGBM builds trees sequentially, where each new tree corrects the errors made by the previous ones. Unlike traditional algorithms that grow trees level by level, LightGBM uses a leaf-wise growth strategy that finds the best split at any leaf, leading to deeper and more accurate trees.

F(x) = F₀(x) + η · h₁(x) + η · h₂(x) + ... + η · hₙ(x)

Where:

  • F(x) = Final prediction
  • η = Learning rate
  • hₙ(x) = Each weak learner (tree)

3. Variables Used in the Model

LightGBM uses a richer set of features compared to simpler models, capturing locality and temporal patterns.

Variable Description
% LED Percentage of LED infrastructure per locality
TOTAL Total public lighting points
LED Number of LED fixtures
Month Month encoded as an ordinal number
Locality Locality encoded as a numeric value

4. Key Innovations of LightGBM

  • Leaf-wise tree growth: Splits the leaf with the highest loss reduction, producing more accurate trees.
  • Gradient-based One-Side Sampling (GOSS): Retains samples with large gradients and randomly drops small gradient samples, speeding up training.
  • Exclusive Feature Bundling (EFB): Bundles mutually exclusive features to reduce dimensionality.
  • Histogram-based splitting: Groups continuous features into bins for faster computation.

5. Application in This Project

LightGBM is applied to estimate failure risk incorporating locality and temporal features, allowing the model to capture seasonal and geographic patterns in public lighting infrastructure.

The model helps identify:

  • Localities with consistently high risk across months
  • Seasonal patterns in infrastructure degradation
  • Geographic clusters of high-risk zones
  • Temporal trends in LED adoption effectiveness

6. Advantages of LightGBM

  • Faster training than XGBoost on large datasets
  • Lower memory consumption
  • Handles categorical features natively
  • Excellent performance on tabular data
  • Supports parallel and GPU learning

7. Model Limitations

  • Prone to overfitting on small datasets without careful tuning
  • Leaf-wise growth can produce unbalanced trees
  • More hyperparameters to tune compared to simpler models
  • Risk labels are estimated rather than obtained from real failures

8. Conclusion

LightGBM offers a powerful and efficient solution for classifying public lighting failure risk. Its ability to incorporate locality and temporal features gives it an edge over simpler models, capturing patterns that single-variable approaches may miss.

Its speed and accuracy make it particularly well-suited for real-world infrastructure monitoring, supporting proactive maintenance decisions across Bogotá's public lighting network.
