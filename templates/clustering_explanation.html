K-Means Clustering Explanation

Unsupervised Learning Model for Public Lighting Pattern Discovery

1. What is K-Means Clustering?

K-Means is an unsupervised machine learning algorithm that groups data points into K clusters based on similarity. Unlike supervised models, it does not require labeled data — it discovers hidden patterns and structures within the dataset on its own.

In this project, K-Means groups Bogotá's localities into clusters based on:

  • LED adoption rates
  • Total luminaire counts
  • Monthly lighting patterns
  • Geographic locality characteristics

2. How Does K-Means Work?

The algorithm follows an iterative process to find optimal cluster assignments:

minimize Σ Σ ||xᵢ - μₖ||²

The process consists of four steps:

  • Step 1 — Initialize: Randomly place K centroids in the feature space.
  • Step 2 — Assign: Assign each data point to the nearest centroid.
  • Step 3 — Update: Recalculate each centroid as the mean of its assigned points.
  • Step 4 — Repeat: Repeat steps 2 and 3 until centroids no longer move significantly.

3. Variables Used in the Model

K-Means uses the richest feature set of all models in this project to capture the full context of each locality.

Variable Description
% LED Percentage of LED infrastructure per locality
TOTAL Total public lighting points
LED Number of LED fixtures
Month Month encoded as an ordinal number
Locality Locality encoded as a numeric value

4. Finding the Optimal K

Two methods are used to determine the best number of clusters:

  • Elbow Method: Plots inertia (within-cluster sum of squares) against K. The optimal K is where the curve bends like an elbow, indicating diminishing returns from adding more clusters.
  • Silhouette Score: Measures how similar each point is to its own cluster compared to other clusters. A score close to 1.0 indicates well-separated, compact clusters.

5. PCA Visualization

Since the model uses 5 features, clusters cannot be visualized directly. Principal Component Analysis (PCA) reduces the data to 2 dimensions while preserving as much variance as possible, allowing clusters to be plotted and interpreted visually.

  • Each point represents one locality-month record.
  • Colors represent different cluster assignments.
  • Black X markers indicate cluster centroids.
  • The percentage shown reflects how much variance is captured by the 2D projection.

6. Application in This Project

K-Means is applied to discover natural groupings in Bogotá's public lighting data without relying on predefined risk labels. Each cluster reveals a distinct infrastructure profile.

Typical cluster profiles may include:

  • High LED adoption, large localities with modern infrastructure
  • Mixed technology zones undergoing gradual modernization
  • Small localities with predominantly legacy technology
  • High-density zones with diverse lighting configurations

7. Advantages of K-Means

  • No labeled data required
  • Simple and computationally efficient
  • Scales well to large datasets
  • Reveals hidden patterns not visible in supervised models
  • Supports infrastructure segmentation for targeted strategies

8. Model Limitations

  • Requires specifying K in advance (mitigated by Elbow and Silhouette methods)
  • Sensitive to feature scaling (addressed with StandardScaler)
  • Assumes spherical, equally-sized clusters
  • Results may vary with different random initializations
  • Cluster labels have no inherent meaning — interpretation requires domain knowledge

9. Conclusion

K-Means Clustering provides a complementary perspective to the supervised models in this project. Rather than predicting a risk label, it discovers natural groupings in the data that reveal structural patterns across Bogotá's localities.

Combined with PCA visualization and hierarchical dendrogram analysis, this approach enables a deeper understanding of the public lighting infrastructure, supporting strategic planning and targeted maintenance allocation across the city.
